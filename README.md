# 數位轉型問卷系統

使用 Bootstrap 5 開發的數位轉型問卷調查系統，包含前台問卷填寫與後台管理功能。

## 專案結構

```
quastionary/
├── frontend/              # 前台頁面（問卷填寫）
│   ├── p01-welcome.html       # 歡迎頁
│   ├── p02-basic-info.html    # 基本資料
│   ├── p03-digital-status.html # 數位現況
│   ├── p04-transformation.html # 轉型經驗
│   └── p05-complete.html      # 完成頁
├── backend/               # 後台頁面（管理系統）
│   ├── a01-login.html         # 登入頁
│   ├── a02-list.html          # 問卷列表
│   ├── a03-detail.html        # 問卷詳情
│   └── a04-dashboard.html     # 儀表板
├── assets/                # 資源檔案
│   ├── css/
│   │   └── custom.css         # 自訂樣式
│   ├── js/                    # JavaScript 檔案
│   └── images/                # 圖片檔案
└── prd.md                 # 產品需求文件
```

## 功能特色

### 前台功能
- ✅ 響應式設計，支援桌面瀏覽器（≥1200px）
- ✅ 4步驟問卷流程，含進度顯示
- ✅ 表單驗證與錯誤提示
- ✅ 資料暫存（localStorage）
- ✅ 隱私權聲明同意機制
- ✅ 完成頁面與確認單

### 後台功能
- ✅ 帳號密碼登入（測試帳號：admin / admin123）
- ✅ 問卷列表與搜尋篩選
- ✅ 問卷詳細資料查看
- ✅ 儀表板統計數據
- ✅ 分頁功能（20筆/頁）
- ✅ 刪除確認機制

## 使用技術

- **前端框架**：Bootstrap 5.3.0
- **CSS**：Custom CSS（遵循設計規範）
- **JavaScript**：原生 JavaScript（無需額外框架）
- **字體**：系統預設字體（PingFang SC, Microsoft YaHei）

## 快速開始

### 1. 開啟前台問卷

直接在瀏覽器中開啟 `frontend/p01-welcome.html`

### 2. 開啟後台管理

直接在瀏覽器中開啟 `backend/a01-login.html`

**測試帳號**：
- 帳號：`admin`
- 密碼：`admin123`

## 頁面說明

### 前台頁面

#### P01 - 歡迎頁
- 系統介紹與說明
- 隱私權政策同意
- 開始填寫按鈕

#### P02 - 基本資料
- 企業基本資料（15個欄位）
- 縣市、產業別選擇
- 聯絡人資訊

#### P03 - 數位現況
- 三大面向評估（行銷/營運/顧客）
- 各項目優先度填寫（1-5）
- 複選題型

#### P04 - 轉型經驗
- 困難點調查（複選）
- 資源需求（最多3項）
- 預期效益與階段評估
- 開放式建議

#### P05 - 完成頁
- 提交成功訊息
- 問卷編號與時間
- 後續事項說明

### 後台頁面

#### A01 - 登入頁
- 帳號密碼登入
- 記住我功能
- 登入狀態檢查

#### A02 - 問卷列表
- 搜尋與篩選功能
- 分頁顯示（20筆/頁）
- 查看與刪除操作
- 匯出 Excel 功能

#### A03 - 問卷詳情
- 完整問卷資料展示
- 列印功能
- 返回列表

#### A04 - 儀表板
- 統計數據卡片
- 縣市分布統計
- 產業別分布統計
- 企業規模統計
- 轉型階段分布
- 困難點與資源需求排名
- 最新問卷列表

## 資料儲存

目前使用 **localStorage** 儲存問卷資料（僅供前端示範）。

實際部署時需要：
1. 建立後端 API
2. 資料庫設計（MySQL、PostgreSQL 等）
3. 整合前後端資料傳輸

## 瀏覽器支援

- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

## 注意事項

1. 本系統為前端靜態頁面，後端功能需另外開發
2. 資料目前儲存在瀏覽器 localStorage，清除瀏覽器資料會遺失
3. 建議使用桌面瀏覽器，最小寬度 1200px
4. 測試資料為模擬資料，實際部署需接入真實 API
